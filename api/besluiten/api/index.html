Service Documentatie¶

De kern van de besluitendatabank is een JSON REST service. Deze kan makkelijk gebruikt worden vanuit een webbrowser in AJAX calls, maar kan ook volledig extern gehanteerd worden.

De volgende endpoints zijn gekend:

Besluiten¶

GET /besluiten¶
GET /besluiten.csv¶

Het ophalen van een verzameling besluiten in csv formaat.

GET /besluiten HTTP/1.1

Accept: text/csv

Query Parameters:

  • provincies – Geef aan voor welke provincies er besluiten moeten gezocht worden.
  • beschermingstypes – Geef aan voor welke beschermingstypes er besluiten moeten gezocht worden.
  • uitvaardigers – Geef aan van welke uitvaardigers er besluiten moeten gezocht worden.
  • rechtsgevolgen – Geef aan voor welke rechtsgevolgen er besluiten moeten gezocht worden.
  • personen – Geef aan voor welke personen besluiten moeten gezocht worden.
  • jaar_ondertekening – Geef aan voor welk ondertekeningsjaar er besluiten moeten gezocht worden.
  • sort – Geef aan op welk veld gesorteerd moet worden. Sorteerrichting kan aangegeven worden met -. bv. id, -onderwerp.
  • start – Vanaf het hoeveelste besluit, de besluiten worden teruggegeven. Standaard is dit 0.
  • aantal – Hoeveel besluiten er teruggegeven worden. Standaard is dit 1000.
Status Codes:
  • 200 OK – Besluiten werden gevonden.

Query:

Stel een vraag. Zal full-text gezocht worden.
Query Parameters:
 
  • onderwerp – Geef aan voor welke onderwerp er beluiten moeten gezocht worden. Er zal tussen de woorden van het onderwerp gezocht worden, rekening houdend met de woordvolgorde.
  • provincies – Geef aan voor welke provincies er besluiten moeten gezocht worden.
  • beschermingstypes – Geef aan voor welke beschermingstypes er besluiten moeten gezocht worden.
  • uitvaardigers – Geef aan van welke uitvaardigers er besluiten moeten gezocht worden.
  • rechtsgevolgen – Geef aan voor welke rechtsgevolgen er besluiten moeten gezocht worden.

  • Range – Kan gebruikt worden om een zekere set van resultaten weer te geven, bv. items=0-24 zal de eerste 25 resultaten opvragen.

  • Content-Range – Laat de gebruiker weten wat de reikwijdte is van de resultaten die worden teruggegeven, bv. items=0-24/306 voor de eerste 25 resultaten uit een totaal van 306.
Status Codes:
  • 200 OK – Besluiten werden gevonden.

        "id": 25
        },
